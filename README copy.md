## **Project Overview**

This project implements a production-style data science pipeline that integrates:

- Earthquake data ingestion and cleaning
- Feature engineering (magnitude binning and region classification)
- Statistical analysis and visualization
- A Streamlit-based interactive dashboard
- Retrieval-Augmented Generation (RAG) for region- and magnitude-specific emergency guidance

The system demonstrates how Generative AI can be integrated into traditional analytical workflows in a reproducible and controlled manner.

## System Architecture

1. Data Ingestion & Cleaning  
2. Feature Engineering  
3. Statistical Analysis & Visualization  
4. Knowledge Base Construction  
5. Embeddings-Based Retrieval  
6. Metadata Filtering (Region + Magnitude)  
7. LLM Protocol Generation

## Retrieval-Augmented Generation (RAG)

The RAG pipeline uses structured knowledge documents derived from multiple sources including FEMA and Ready.gov emergency preparedness guidance. Emergency preparedness guidance was structured into semantically organized text documents categorized by:

- Region Type (Urban / Rural / General)
- Magnitude Severity (Light / Moderate / Strong)
- Action Stage (Before / During / After)

Knowledge Base files are stored as .txt in /kb.
Files are chunked by ## headers when present; otherwise they are chunked by paragraphs.
For best retrieval quality and clean citations, we recommend using ## section headers and a dedicated ## Sources section.

## Ethical and Practical Considerations

- AI-generated guidance is grounded in authoritative (US government) public safety sources.
- The system does not replace official emergency management agencies.
- Retrieval-based design mitigates hallucination risk.
- Outputs should be interpreted as informational, not medical or legal advice.

## How to Run This Project

1.  Clone the repository:
```
git clone (https://github.com/KatieKimberling/ADS-509-Team5-Final-Project.git)
cd ADS-509-Team5-Final-Project
```

2.  Create & Activate Environment:  
We recommend a fresh Conda environment to avoid dependency conflicts.
```
conda create -n ads509_eq python=3.11
conda activate ads509_eq
```

Install required packages:
```
pip install -r requirements.txt
```

If requirements.txt is not provided:
```
pip install streamlit pandas numpy scikit-learn openai python-dotenv pyarrow
```

3.  Add Your OpenAI API Key by creating a .env file in the project root directory:
```
OPENAI_API_KEY=your_api_key_here
```

4.  Generate Required Artifacts (One-Time Setup):  
Before launching Streamlit, ensure the following files exist inside /outputs:
-   clean_earthquakes.parquet or clean_earthquakes.csv
-   stat_pack.json
-   figure_paths.json. 
These are generated by running the main project notebook: Earthquakes.ipynb
-   Confirm that the /outputs folder contains the files above.

5. Launch the Streamlit App:  
From the project root type: 
```
streamlit run app.py
```
The app will automatically open in your browser.

## Using the Application
## **Please note the two main tabs at the top of the screen**

**📊 Tab 1: Briefing Dashboard**. 
Generates an AI-powered executive earthquake report. 
Uses:
-   stat_pack.json
-   Top N largest magnitude events
-   User can regenerate the briefing using sidebar controls
-   Downloadable as single-file HTML (exportable to PDF)

**🧠 Tab 2: Protocol Generator (RAG)**. 
This feature generates earthquake response protocols using a team-generated knowledge base.  
How it works:
-   Retrieves top_k passages from /kb folder
-   The top_k slider controls how many knowledge-base chunks are retrieved and used as evidence.
-   Lower (2–3): tighter, more focused protocol
-   Moderate (4–6): balanced coverage
-   Higher (7–10): broader but may introduce noise
-   (Recommended: 3–5)

-   Sends only those passages to GPT-5.2
-   Model generates a structured JSON protocol
-   The output is saved with full audit trail for transparency
